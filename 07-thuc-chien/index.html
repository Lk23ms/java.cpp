MISSION 01 · FORGE ITEM

Potion Ring — Nhẫn hiệu ứng

DỄ

Tạo một item mới tên "Nhẫn Cuồng Phong": khi người chơi giữ chuột phải, nhân vật nhận hiệu ứng Speed II trong 5 giây, item có cooldown 10 giây.

  • Đăng ký item bằng DeferredRegister (ôn lại Chương 03).
  • Override phương thức xử lý khi dùng item (click chuột phải).
  • Áp dụng MobEffectInstance và cooldown bằng ItemCooldowns.
Gợi ý 1 — bắt đầu từ đâu?
Item tùy chỉnh hành vi cần override use() trong class kế thừa Item. Xem lại cách đăng ký item ở Chương 03 mục 3.4.
Gợi ý 2 — API cần dùng
Dùng player.addEffect(new MobEffectInstance(MobEffects.MOVEMENT_SPEED, 100, 1)) (100 tick = 5 giây, amplifier 1 = Speed II). Đặt cooldown bằng player.getCooldowns().addCooldown(this, 200) (200 tick = 10 giây).
Gợi ý 3 — gần đáp án
Trả về InteractionResultHolder.success(itemStack) ở cuối hàm use() để game biết hành động đã thành công (đồng bộ animation).
✅ Đáp án tham khảo 
public class WindRingItem extends Item {
    public WindRingItem(Properties props) { super(props); }

    @Override
    public InteractionResultHolder<ItemStack> use(Level level, Player player, InteractionHand hand) {
        ItemStack stack = player.getItemInHand(hand);
        if (!level.isClientSide && !player.getCooldowns().isOnCooldown(this)) {
            player.addEffect(new MobEffectInstance(MobEffects.MOVEMENT_SPEED, 100, 1));
            player.getCooldowns().addCooldown(this, 200);
        }
        return InteractionResultHolder.success(stack);
    }
}
MISSION 02 · SPIGOT PLUGIN

Auto-Heal có Cooldown

DỄ

Viết lệnh /heal: hồi đầy máu người chơi, nhưng chỉ dùng được 1 lần mỗi 60 giây. Nếu dùng sớm hơn, báo còn bao nhiêu giây nữa mới dùng lại được.

  • Lưu thời điểm dùng lệnh gần nhất theo từng người chơi (UUID → timestamp).
  • So sánh với thời gian hiện tại mỗi lần gọi lệnh.
Gợi ý 1
Dùng Map<UUID, Long> lưu System.currentTimeMillis() của lần cuối dùng lệnh (ôn lại Chương 02 mục Collections).
Gợi ý 2
Tính chênh lệch: long elapsed = (now - lastUsed) / 1000; nếu elapsed < 60 thì báo còn 60 - elapsed giây.
Gợi ý 3
Đừng quên cập nhật lại timestamp ngay sau khi hồi máu thành công, nếu không cooldown sẽ không bao giờ được set lần đầu.
✅ Đáp án tham khảo 
private final Map<UUID, Long> lastHeal = new HashMap<>();

@Override
public boolean onCommand(CommandSender sender, Command cmd, String label, String[] args) {
    if (!(sender instanceof Player player)) return false;

    long now = System.currentTimeMillis();
    long last = lastHeal.getOrDefault(player.getUniqueId(), 0L);
    long elapsedSec = (now - last) / 1000;

    if (elapsedSec < 60) {
        player.sendMessage("Chờ thêm " + (60 - elapsedSec) + " giây nữa!");
        return true;
    }

    player.setHealth(20.0);
    lastHeal.put(player.getUniqueId(), now);
    player.sendMessage("Đã hồi đầy máu!");
    return true;
}
MISSION 03 · FABRIC CLIENT / DEV TOOL

Hitbox Debug Visualizer

TRUNG BÌNH

Viết mod client vẽ khung viền hitbox của các entity xung quanh khi người chơi giữ phím F6 — công cụ debug hữu ích khi tự phát triển mod/entity, giống tính năng "Show Hitboxes" (F3+B) nhưng do bạn tự làm.

  • Đăng ký keybind (ôn Chương 05 mục 5.4).
  • Duyệt danh sách entity trong tầm nhìn, lấy getBoundingBox().
  • Vẽ outline bằng WorldRenderEvents (chỉ vẽ, không đổi collision thật).
Gợi ý 1
Mỗi Entity có sẵn entity.getBoundingBox() trả về một Box (AABB) — đây chính là hitbox thật, bạn chỉ cần vẽ lại nó, không cần tự tính toán.
Gợi ý 2
Dùng hook WorldRenderEvents.LAST (Fabric API) để vẽ sau khi world đã render xong, tránh bị che khuất. Vẽ bằng VertexConsumer với chế độ LINES.
Gợi ý 3
Đây là công cụ chỉ hiển thị — không gọi bất kỳ hàm nào làm thay đổi boundingBox gốc của entity. Nếu thấy mình đang sửa kích thước hitbox thật thay vì chỉ vẽ nó, đó là dấu hiệu đi sai hướng bài này.
✅ Đáp án tham khảo (rút gọn) 
private static boolean showHitboxes = false;

@Override
public void onInitializeClient() {
    KeyBinding toggleKey = KeyBindingHelper.registerKeyBinding(new KeyBinding(
        "key.examplemod.togglehitbox", InputUtil.Type.KEYSYM, GLFW.GLFW_KEY_F6, "category.examplemod.general"));

    ClientTickEvents.END_CLIENT_TICK.register(client -> {
        while (toggleKey.wasPressed()) showHitboxes = !showHitboxes;
    });

    WorldRenderEvents.LAST.register(context -> {
        if (!showHitboxes) return;
        MinecraftClient client = MinecraftClient.getInstance();
        for (Entity e : client.world.getEntities()) {
            if (e == client.player) continue;
            // Vẽ box wireframe từ e.getBoundingBox() tại vị trí camera-relative
            WorldRenderer.drawBox(context.matrixStack(), context.consumers().getBuffer(RenderLayer.getLines()),
                e.getBoundingBox().offset(-context.camera().getPos().x, -context.camera().getPos().y, -context.camera().getPos().z),
                1f, 0.2f, 0.9f, 1f); // màu neon tím cho dễ nhìn :)
        }
    });
}

Code rút gọn để minh họa luồng chính — bản đầy đủ cần import đúng class Fabric API tương ứng phiên bản Minecraft bạn dùng.

MISSION 04 · SPIGOT ANTI-CHEAT

Reach Hack Detector

KHÓ

Đây là bài "anticheat" đúng nghĩa: viết plugin phát hiện người chơi đánh trúng entity khác từ khoảng cách bất thường (reach hack) — kỹ năng nền tảng của mọi anti-cheat PvP thật.

  • Lắng nghe sự kiện EntityDamageByEntityEvent.
  • Tính khoảng cách giữa attacker và victim tại thời điểm đánh trúng.
  • Khoảng cách hợp lệ trong Minecraft vanilla ~3 block (survival) — nếu vượt ngưỡng (ví dụ 4.2 block) nhiều lần liên tiếp → cảnh báo admin.
Gợi ý 1
event.getDamager() và event.getEntity() đều có getLocation() → dùng loc1.distance(loc2) để tính khoảng cách.
Gợi ý 2
Đừng cảnh báo ngay ở lần vượt ngưỡng đầu tiên — độ trễ mạng (lag) có thể gây sai số. Đếm số lần vi phạm liên tiếp trong một khoảng thời gian ngắn (ví dụ 5 lần trong 10 giây) trước khi báo, để giảm false positive.
Gợi ý 3
Dùng lại kỹ thuật Map<UUID, Integer> đếm vi phạm (giống Mission 02), và Bukkit.getScheduler().runTaskLater(...) để tự reset bộ đếm sau X giây.
✅ Đáp án tham khảo 
private final Map<UUID, Integer> violations = new HashMap<>();
private static final double MAX_REACH = 4.2; // block

@EventHandler
public void onHit(EntityDamageByEntityEvent event) {
    if (!(event.getDamager() instanceof Player attacker)) return;

    double distance = attacker.getLocation().distance(event.getEntity().getLocation());
    if (distance > MAX_REACH) {
        int count = violations.merge(attacker.getUniqueId(), 1, Integer::sum);

        if (count >= 5) {
            Bukkit.getLogger().warning("[AntiCheat] " + attacker.getName()
                + " nghi ngờ dùng Reach Hack (khoảng cách " + String.format("%.2f", distance) + ")");
            violations.remove(attacker.getUniqueId());
            // TODO: thông báo cho admin online, hoặc ghi log vào file để review sau
        }

        // Tự reset bộ đếm sau 10 giây nếu không vi phạm thêm
        Bukkit.getScheduler().runTaskLater(plugin, () ->
            violations.remove(attacker.getUniqueId()), 20L * 10);
    }
}

Anti-cheat thật sự (như NoCheatPlus, Grim) còn kiểm tra thêm góc nhìn (angle), độ trễ ping, và nhiều pattern khác — đây là bản nền tảng để bạn hiểu nguyên lý trước khi đọc source code của họ.

MISSION 05 · DEBUG CHALLENGE

Bắt lỗi: Plugin Kinh Tế Bị Lỗi

TRUNG BÌNH

Đoạn code dưới đây có 2 lỗi khiến người chơi có thể chuyển tiền âm hoặc tự nhân đôi tiền của mình. Tìm và sửa cả hai lỗi.

public boolean pay(Player from, Player to, int amount) {
    int fromBalance = balances.get(from.getUniqueId());
    int toBalance = balances.get(to.getUniqueId());

    balances.put(from.getUniqueId(), fromBalance - amount);
    balances.put(to.getUniqueId(), toBalance + amount);
    return true;
}
Gợi ý 1
Thử gọi pay(steve, steve, 100) (chuyển cho chính mình) hoặc pay(steve, alex, -100) — điều gì xảy ra với số dư?
Gợi ý 2
Lỗi 1: hàm không kiểm tra amount > 0 và fromBalance >= amount trước khi trừ tiền.
Gợi ý 3
Lỗi 2: nếu from và to là cùng 1 người, 2 dòng put() sẽ ghi đè nhau sai — dòng thứ 2 dùng lại toBalance đã cũ (chưa trừ), gây nhân tiền ảo.
✅ Đáp án tham khảo 
public boolean pay(Player from, Player to, int amount) {
    if (amount <= 0) return false; // chặn số âm / bằng 0

    int fromBalance = balances.get(from.getUniqueId());
    if (fromBalance < amount) return false; // chặn không đủ tiền

    balances.put(from.getUniqueId(), fromBalance - amount);

    // Đọc lại số dư người nhận SAU KHI đã cập nhật người gửi,
    // để xử lý đúng trường hợp from == to
    int toBalance = balances.get(to.getUniqueId());
    balances.put(to.getUniqueId(), toBalance + amount);

    return true;
}
MISSION 06 · FORGE WORLDGEN

Thêm Quặng Mới Vào Thế Giới

KHÓ

Thêm 1 loại quặng "Ruby Ore" sinh ra tự nhiên trong hang động ở độ cao Y từ 0 đến 40, độ hiếm tương đương Diamond Ore.

  • Đăng ký Block quặng (ôn Chương 03 mục 3.5).
  • Tạo ConfiguredFeature mô tả cách quặng sinh ra (kích thước vein).
  • Tạo PlacedFeature mô tả nơi đặt (độ cao, tần suất) và đăng ký vào biome.
Gợi ý 1
Forge dùng hệ thống Feature → ConfiguredFeature → PlacedFeature theo thứ tự đó — mỗi lớp thêm một tầng cấu hình. Đừng nhảy thẳng vào PlacedFeature khi chưa có ConfiguredFeature.
Gợi ý 2
Có thể copy gần như nguyên cấu hình sinh Diamond Ore trong code gốc Minecraft (net.minecraft.data.worldgen) rồi đổi Block target — không cần viết từ số 0.
Gợi ý 3
Đăng ký PlacedFeature vào biome bằng cách dùng BiomeLoadingEvent và kiểm tra event.getCategory() == Biome.BiomeCategory.??? để chỉ thêm vào loại biome phù hợp (ví dụ chỉ hang động/núi).
✅ Đáp án tham khảo (khung sườn) 
public static final ConfiguredFeature<?, ?> RUBY_ORE = Feature.ORE.configured(
    new OreConfiguration(List.of(
        OreConfiguration.target(OreFeatures.STONE_ORE_REPLACEABLES, RUBY_BLOCK.get().defaultBlockState())
    ), 6) // kích thước vein ~ giống Diamond
);

public static final PlacedFeature RUBY_ORE_PLACED = RUBY_ORE.placed(
    CountPlacement.of(4),                     // 4 vein mỗi chunk
    InSquarePlacement.spread(),
    HeightRangePlacement.uniform(VerticalAnchor.absolute(0), VerticalAnchor.absolute(40)),
    BiomeFilter.biome()
);

@SubscribeEvent
public static void addFeature(BiomeLoadingEvent event) {
    if (event.getCategory() == Biome.BiomeCategory.EXTREME_HILLS
     || event.getCategory() == Biome.BiomeCategory.UNDERGROUND) {
        event.getGeneration().getFeatures(GenerationStep.Decoration.UNDERGROUND_ORES)
            .add(() -> RUBY_ORE_PLACED);
    }
}
